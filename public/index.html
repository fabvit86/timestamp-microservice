Timestamp Microservice

USAGE:

  1. Pass a string as a parameter in the URL, the API will check whether the string contains either a unix timestamp or a natural language date.
  2. If it does, it will return both the Unix timestamp and the natural language form of that date in a JSON object.
  3. If it does not contain a date or Unix timestamp, it will return null for those properties in the JSON object.

EXAMPLES:

  • Passing /1496793600 or /Wed June 07, 2017 after the root URL will return {"unix":1496793600,"natural":"Wed Jun 07 2017"}.
  • Passing a non-Date string will return {"unix":null,"natural":null}.
